#  Vaxis Studio
### *XR Pipeline — v1.1.0*

> 「NPC挙動のscriptコーディング工数を、極限まで省略する。」
>
> あなたはもう、3DCGの動きをコードで書き下す必要はない。
> テキストを入力するだけで、数千体の意思を持つエンティティがUnityに降り立つ。

---

##  このツールが実現すること

従来のXR開発において、NPCに「パトロール」「話しかける」「敵を探す」などの行動を実装するには、スクリプトの設計・実装・デバッグテストが必要でした。
キャラクターごとに異なるコードが散在し、チームの生産性を著しく圧迫していました。

Vaxisは、その工数を極限まで削減します。

| Before（従来） | After（Vaxisを使えば） |
|:---|:---|
| エンジニアかクリエイターがNPC挙動を設計 | クリエイターがテキストで挙動を記述 |
| キャラクターごとに個別のコーディング | 「守護妖怪、神社周囲50mを巡回」といった指示を入力で完結 |
| モデリングからUnityの統合まで数週間 | 「Text-to-3D」によるワンクリック自動生成・デプロイ |
| スケール困難（100体→リアリティを求めるなら個別に難易度急増エンジニアリングコストも爆増） | 数千体の量産に同じコストで対応可能 |

##  パイプライン全体像
『従来の手法』
所要時間: 数週間〜（エンジニアの稼働が必須）

ワークフロー:
クリエイターが数日かけてモデリングとテクスチャ作成を行い、FBXデータをエンジニアに渡す。
エンジニアがAnimatorやステートマシンを設定し、数日かけてNPCの挙動条件をハードコーディングする。
挙動のリテイクが発生するたびに、エンジニアが挙動を直接書き直して再ビルドを行う。

課題: エンジニアのコーディング工数が膨大になり、微調整のたびに開発が停滞し疲弊する。




『✨本ツール導入後✨』

所要時間: 数分（完全ゼロコーディング）

ワークフロー:
クリエイターがBlender上でAIテクスチャを即時生成し、NPCの性格や指示を「テキスト」で入力してエクスポートする。
Unity側が転送されたデータ（FBX＋JSON）を自動検知し、数秒でシーンへNPCを自動配置する。
アタッチされたAIがテキスト指示を読み込み、即座に自律行動を開始する。
リテイクが発生しても、クリエイターがBlender上でテキストを書き直すだけで完了する。

成果: エンジニアの実装工数が限りなく少なくなり、別の詳細な調整に手が回るようになります。
クリエイター単独で意思を持つNPCの量産・統合が可能なので作業工数の大幅短縮が可能です。

---### STEP 1: Blender側のセットアップ

#### 1-1. Blender 5.1 Extension のパッケージビルド

以下のコマンドを実行して、配布用の ZIP ファイルを生成します。

```bash
python build_package.py
```

`IsekaiAssetStudio_v1.0.0.zip` 
がカレントディレクトリに出力されます。

#### 1-2. Blender へのインストール

1. Blender 5.1 を起動します。
2. 上部メニューからEdit → Preferences → Extensionsを開きます。
3. 右上の「∨」ドロップダウン → Install from Disk をクリックします。

   > ⚠️ Blender 5.1の注意点
   > 旧来の「Add-ons」メニューからではなく、必ず **「Extensions」** メニューを使用してください。

4. 生成した `IsekaiAssetStudio_v1.0.0.zip` を選択します。
5. リストに表示される「Isekai Asset Studio」のチェックボックスにチェックを入れます。

#### 1-3. BYOK (APIキー) の設定

本ツールはベースとして「Tripo3D API」または「Meshy API」を用いた Text-to-3D 自動生成をサポートしています。
随時追加予定です。任意のAPIにも対応します。

1. 引き続き Blender の Edit → Preferences 画面を開きます。
2. 左側のメニューから Add-ons タブを選択し、検索欄で「Isekai」と検索します。
3. リストから Isekai Asset Studio を開き、設定（Preferences）パネルを展開します。
4. ご自身で取得した Tripo3D API Key または Meshy API Key を入力して保存します。

---

### STEP 2: Unity側のセットアップ

#### 2-1. Unity 6 プロジェクトの準備

1. Unity Hub から Unity 6 (6000.x.x) のプロジェクトを作成。
2. 本リポジトリ内の `Unity_Package/Assets/IsekaiSystem/` フォルダを Unity プロジェクトの `Assets/` 以下にコピー。

#### 2-2. Sentis 2.0 パッケージなど必須依存の導入

1. `Unity Editor → Window → Package Manager` を開く。
2. 左上の「+」ボタンから `Add package by name...` を選択し、`com.unity.sentis` を入力してインストール。

#### 2-3. 自動検知フォルダの作成

1. Unity の `Assets/` 階層直下に `AutoImport/` という名前のフォルダを作成します（ここがBlenderからのエクスポート先になります）。

##  起動方法と使い方

3D Viewport を開き、キーボードの `N` キー を押して右サイドバーを表示します。
「Isekai」タブ が拡張機能のメインパネルです。

### 1. 3Dモデルの自動生成 (Text-to-3D)
1. 操作モードを 「3Dモデル生成」に設定。
2. 生成バックエンド（Tripo3D または Meshy）を選択。
3. 3D Prompt 欄に作りたいキャラクターを英語で入力します。翻訳機能はまだ未実装です。（例: A low-poly ancient golem with glowing eyes）
※すでに翻訳機能は様々なアドオンが存在するので優先しませんでした。あれば便利だと思うので開発はするつもりです辞書登録などの機能も実装予定なので特有の用語も対応できます。
4. 「3Dモデルを自動生成」 ボタンを押します。
   *※バックグラウンドでAPIが走り、完了すると自動的にBlenderのシーン中央にキャラクターが配置されます。*

### 2. 性格・行動の付与と Unity へのデプロイ
1. モデル（自動生成したもの、または自分で作成したメッシュ）を選択モードにしたままにします。
2. AIエージェント定義セクションに以下を入力します:
   - ロール: `守護ゴーレム`
   - 行動指示: `マスターの半径5m以内を追従し、近づく敵には赤い光を放って威嚇する。`
3. 操作モードを 「Unityへ出力」 に切り替えます。
4. 「 Unity へデプロイ」 ボタンを押します。

### 3. Unity での自動起動確認
Blenderから書き出されると、Unityが自動的に `.json` と `.fbx` を検知します。
- `AutoImport/` フォルダ内でモデルが自動セットアップされ、シーンにプレハブが配置されます。
- `IsekaiAgentController` がアタッチされ、1秒ごとにコンソールへエージェントの自律思考状態が出力されるようになります。
